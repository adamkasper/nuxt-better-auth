---
title: SSR Sessions Guide
description: Avoid session flashes during server-side rendering and hydration.
---

Use this pattern when a page needs session data during server-side rendering (SSR) without waiting for hydration.

```vue [pages/dashboard.vue]
<script setup lang="ts">
definePageMeta({ auth: 'user' })
const { user, ready, fetchSession } = useUserSession()

if (import.meta.server) {
  // On SSR, call fetchSession with request cookies to avoid a flash
  await fetchSession({ headers: useRequestHeaders(['cookie']) })
}
</script>

<template>
  <BetterAuthState>
    <template #default>
      <p v-if="ready">Welcome {{ user?.name }}</p>
    </template>
    <template #placeholder>
      <p>Loading sessionâ€¦</p>
    </template>
  </BetterAuthState>
</template>
```

::note
Without server-side session initialization, the page briefly shows a loading state before hydration completes. Fetching the session on the server prevents this flash.
::
