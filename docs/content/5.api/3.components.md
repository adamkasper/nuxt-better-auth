## `<BetterAuthState>`

Renders once session hydration is complete (`ready === true`) and exposes state via slots.

::note
This is currently the only component provided. Additional components may be added in future versions.
::

```vue [components/AuthWrapper.vue]
<BetterAuthState>
  <template #default="{ loggedIn, user, signOut }">
    <p v-if="loggedIn">Hi {{ user?.name }}</p>
    <button v-else @click="navigateTo('/login')">Sign in</button>
  </template>
  <template #placeholder>
    <p>Loading…</p>
  </template>
</BetterAuthState>
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `ready` | `boolean` | `true` after initial session fetch completes |
| `user` | `AuthUser \| null` | Current user object or `null` |
| `session` | `AuthSession \| null` | Current session object or `null` |
| `loggedIn` | `boolean` | `true` if user is authenticated |
| `fetch` | `() => Promise<void>` | Manually refresh session |
| `clear` | `() => void` | Clear local session state |

### Slots

| Slot | Props | When it renders |
| :--- | :--- | :--- |
| `default` | `{ loggedIn, user, session, signOut }` | When the session is hydrated (`ready === true`). |
| `placeholder` | — | While the session is still hydrating. |

::callout{icon="i-lucide-info" to="/api/composables"}
Need `signIn` / `signUp`? Use `useUserSession()` directly.
::
